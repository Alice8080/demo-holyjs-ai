# Демо Prompt API

Интерактивная демонстрация [встроенного Prompt API](https://developer.chrome.com/docs/ai/built-in) в Chrome. Модель **Gemini Nano** работает локально на устройстве — без обращения к облачным серверам. Интерфейс на русском языке.

Демо позволяет отправлять промпты, наблюдать потоковую генерацию ответов, отслеживать расход контекстного окна и анализировать производительность модели в реальном времени.

Форк от [демо Prompt API](https://github.com/GoogleChromeLabs/web-ai-demos/tree/main/prompt-api-playground)

## Требования

- **Google Chrome** с поддержкой Prompt API (встроенный Gemini Nano)
- **HTTPS** — API доступен только в защищённом контексте (`https://` или `localhost`)
- Если API недоступен, на странице появится сообщение со ссылкой на [программу раннего доступа Chrome AI](https://goo.gle/chrome-ai-dev-preview-join)

> Демо также совместимо с [polyfill Prompt API](../prompt-api-polyfill/) (например, на базе Transformers.js). В этом случае метрики и режим обработки отображаются иначе — см. раздел [Метрики производительности](#метрики-производительности).

## Структура проекта

| Файл | Назначение |
| --- | --- |
| `index.html` | Разметка страницы: форма ввода, таблицы статистики и метрик, область диалога |
| `script.js` | Вся логика: сессия модели, стриминг, метрики, шаринг ссылок |
| `style.css` | Стили: «пузыри» диалога, таблицы, тёмная/светлая тема |
| `package.json` | Скрипты запуска и форматирования |

Внешние зависимости подключаются через CDN прямо из `script.js`:

- **marked** — рендер Markdown в HTML
- **DOMPurify** — санитизация HTML перед вставкой в DOM

## Возможности

### Ввод промпта

- Текстовое поле для промпта
- **Enter** — отправить, **Shift+Enter** — перенос строки
- Кнопка **«Отправить промпт»** — запускает потоковую генерацию
- Под полем ввода отображается **стоимость промпта в токенах** (обновляется по мере набора текста)

### Диалог

- Промпты и ответы отображаются в виде «речевых пузырей» (синий — пользователь, жёлтый — модель)
- Ответ приходит **потоком** (`promptStreaming`) — текст появляется по мере генерации
- Markdown в ответах рендерится в HTML (заголовки, списки, код и т.д.)
- Блок **«Исходный ответ»** (`<details>`) показывает сырой текст без форматирования

### Статистика сессии

Таблица отражает состояние контекстного окна текущей сессии:

| Поле | Описание |
| --- | --- |
| **Использовано токенов** | Сколько токенов уже занято в сессии (`contextUsage`) |
| **Осталось токенов** | Свободное место в контекстном окне |
| **Всего токенов** | Размер контекстного окна (`contextWindow`) |

Код поддерживает несколько поколений API: `contextWindow` / `contextUsage`, а также устаревшие `inputQuota` / `inputUsage` и `maxTokens` / `tokensSoFar`.

### Метрики производительности

Панель метрик помогает оценить, как модель работает на конкретном устройстве.

| Метрика | Что показывает | Как считается |
| --- | --- | --- |
| **Использование памяти** | Текущий объём памяти и приблизительный лимит | `performance.memory.usedJSHeapSize` и `jsHeapSizeLimit` (или `navigator.deviceMemory`) |
| **Время последнего ответа** | Длительность последней генерации | `performance.now()` от вызова `promptStreaming` до получения последнего чанка |
| **Токенов в секунду** | Скорость генерации | Разница `contextUsage` до и после ответа, делённая на время |
| **Выбранная модель** | Имя загруженной модели | `Gemini Nano` (нативный API) или `smollm2-135m-instruct` (polyfill) |
| **Время загрузки модели** | Инициализация сессии | Время выполнения `LanguageModel.create()` |
| **Пиковое использование памяти** | Максимум с момента запуска | Наибольшее значение `usedJSHeapSize` за сессию |
| **Режим обработки** | Где выполняется инференс | «Основной поток» или «Web Worker (фоновый поток)» |

**История времени ответа** — список длительностей всех предыдущих запросов (новые сверху). Удобна для сравнения производительности на разных промптах.

> Метрики памяти отражают потребление JS-кучи страницы, а не точный объём VRAM/GPU, который использует модель внутри браузера. Prompt API не предоставляет встроенных API для замера скорости — все показатели вычисляются на стороне демо.

### Сброс сессии

Кнопка **«Сбросить сессию»**:

1. Очищает поле ввода и историю диалога
2. Сбрасывает метрики ответов (время, история, токены/с)
3. Вызывает `session.destroy()` и создаёт новую сессию

Системный промпт при этом сохраняется.

### Шаринг ссылок

Кнопка **«Копировать ссылку»** формирует URL с параметрами:

- `?prompt=...` — текст промпта (URL-encoded)
- `&highlight=...` — выделенный фрагмент ответа (если текст был выделен мышью)

При открытии такой ссылки демо автоматически отправляет промпт и, при наличии `highlight`, показывает **проблемный фрагмент** — удобно для репортов некорректных ответов.

## Как это работает

### Инициализация

При загрузке страницы:

1. Проверяется наличие `LanguageModel` в `globalThis`
2. Создаётся сессия через `LanguageModel.create()` с системным промптом:
   ```
   Вы отзывчивый и дружелюбный помощник.
   ```
3. Заполняются статические метрики (модель, режим обработки, время загрузки)

### Генерация ответа

```
Пользователь → promptStreaming(prompt) → AsyncIterable<string>
                    ↓
              Каждый чанк → marked + DOMPurify → DOM
                    ↓
              После завершения → обновление метрик и статистики
```

Стриминг реализован через `for await...of`. Каждый чанк от API содержит **полный накопленный текст**, поэтому код вычисляет дельту относительно предыдущего чанка:

```js
const newChunk = chunk.startsWith(previousChunk)
  ? chunk.slice(previousChunk.length)
  : chunk;
```

### Оценка стоимости промпта

При вводе текста вызывается один из методов (в порядке приоритета):

- `session.countPromptTokens(value)` — устаревший
- `session.measureContextUsage(value)` — актуальный
- `session.measureInputUsage(value)` — промежуточный

Результат отображается рядом с кнопками с правильным склонением: «1 токен», «2 токена», «5 токенов».

### Безопасность

- Страница работает только по HTTPS (принудительный редирект в `<head>`)
- HTML-ответы проходят через **DOMPurify** перед вставкой в DOM
- Origin Trial token в `<meta>` включает экспериментальную функцию multimodal input для `chrome.dev:443`
